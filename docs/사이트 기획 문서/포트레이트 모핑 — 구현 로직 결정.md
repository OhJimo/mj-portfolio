## 1. 컴포넌트 아키텍처 — Provider 패턴

### 구조
```
<App>
  <MorphProvider>          ← context 제공
    <Header />
    <main ref={mainRef}>
      <IntroSection />     ← introRef 등록
      <AboutSection />     ← aboutRef 등록
      <ExperienceSection />
      ...
      <MorphingPortrait /> ← context 소비, 카드 렌더
    </main>
    <Footer />
  </MorphProvider>
</App>
```

### 이유
- Intro/About 섹션과 floating card는 DOM 트리상 거리가 멀어 props 드릴링 비효율
- Provider가 자연스럽게 anchor refs와 main ref를 공유
- 4차 시도에서도 동일 패턴 사용 → 검증된 구조

### Provider가 노출하는 것
```ts
{
  introRef:  RefObject<HTMLElement | null>
  aboutRef:  RefObject<HTMLElement | null>
  mainRef:   RefObject<HTMLElement | null>
  isActive:  boolean   // 데스크톱 + reduce-motion 미설정 시 true
}
```

### 등록 패턴
- 섹션은 자기 anchor 요소에 `ref={introRef}` 직접 부착
- floating card는 `useContext(MorphContext)`로 모두 읽음

## 2. 카드 포지셔닝 — `position: absolute` (in `<main>`)

```tsx
<main ref={mainRef} className="relative">
  ...
  <MorphingPortrait />  {/* absolute, z-30 */}
</main>
```

### 왜 absolute인가
- anchor 좌표를 `<main>` top-left 기준으로 계산하면 scroll-stable
- 카드는 `<main>`과 함께 스크롤 → 자연스럽게 페이지 따라감
- transform은 progress 기반 보간만 책임

### `fixed` 대안 폐기
- viewport 좌표는 매 프레임 재계산 필요
- scroll 이벤트마다 RAF 갱신 → 비용 증가
- 채택 안 함

---

## 3. 좌표 계산 — `getBoundingClientRect` 차이

```tsx
function calcCoords(): MorphCoords | null {
  const main  = mainRef.current?.getBoundingClientRect()
  const intro = introRef.current?.getBoundingClientRect()
  const about = aboutRef.current?.getBoundingClientRect()
  if (!main || !intro || !about) return null

  return {
    sx: intro.left - main.left + (intro.width  - CARD_W) / 2,
    sy: intro.top  - main.top  + (intro.height - CARD_H) / 2,
    ex: about.left - main.left + (about.width  - CARD_W) / 2,
    ey: about.top  - main.top  + (about.height - CARD_H) / 2,
    introDocY: intro.top + window.scrollY,
    aboutDocY: about.top + window.scrollY,
  }
}
```

### 측정 시점
- 마운트 직후
- `resize` 이벤트
- 폰트 로드 후 1회 (텍스트 폭 변화로 anchor 위치 변동 가능)

### 측정 안 하는 시점
- 스크롤 중 — 좌표 자체는 scroll-stable이므로 불필요

### 좌표 의미
- **`(sx, sy)`**: `<main>` 기준 intro anchor 중심 — 카드의 시작 위치
- **`(ex, ey)`**: `<main>` 기준 about anchor 중심 — 카드의 끝 위치
- **`introDocY`/`aboutDocY`**: 문서 절대 좌표 — 진행도 매핑용

## 4. 진행도 0~1 정의 — Anchor center to Anchor center (Option A)

### 정의
- **progress 0**: intro anchor가 viewport 정중앙에 위치한 시점
- **progress 1**: about anchor가 viewport 정중앙에 위치한 시점

### 매핑
```ts
const startScrollY = introDocY - viewportHeight / 2
const endScrollY   = aboutDocY - viewportHeight / 2
const progress = clamp((scrollY - startScrollY) / (endScrollY - startScrollY), 0, 1)
```

### 우리 레이아웃에서 실제 윈도우
- intro anchor 약 44vh (Intro 섹션 중앙)
- about anchor 약 123vh (About 섹션의 portrait)
- viewport 100vh
- → progress 0 시점: `scrollY = -6vh` (사실상 페이지 시작)
- → progress 1 시점: `scrollY = 73vh`
- → **morph 윈도우 ≈ 73vh** (약 0.73 viewport 분량의 스크롤)

### 사용자 체감
- 페이지 진입 직후부터 카드가 미세하게 움직이기 시작
- intro 섹션이 떠나는 동안 회전 깊어짐
- about 섹션 진입과 동시에 카드 안착 완료

### 왜 Option A인가
1. SSOT의 "동시 점진적 이동" 정신과 일치
2. 카드가 항상 보이고 항상 반응 → 스크롤이 곧 모핑
3. 레퍼런스(Joseph Robert) 인상도 여정형
4. 빠른 스크롤에도 카드가 뭔가 변하고 있음을 느낌

### 폐기된 대안
- **Option B (섹션 boundary 기반)**: 섹션 안에서는 카드 정지 → 스크롤 반응성 떨어짐
- **Option C (tight window)**: 너무 빠른 morph → 어색함

### A로 가다 안 어울리면
- intro 안에서 카드가 너무 일찍 회전 시작한다고 느끼면 → B로 변경
- 변경 비용 작음 (스크롤 매핑 함수의 시작/끝 값만 변경)

## 5. Wrapper 구조 — 3 레이어 분리

```tsx
<motion.div                            // 1) 이동 wrapper
  className="absolute z-30 pointer-events-none"
  style={{
    left: 0, top: 0,
    width: CARD_W, height: CARD_H,
    x, y,                              // useTransform 결과
    opacity: ready ? 1 : 0,
  }}
>
  <motion.div                          // 2) 회전 wrapper
    className="size-full"
    style={{ rotateY, rotateZ, transformOrigin: "center" }}
  >
    <motion.img                        // 3a) intro 이미지
      src={profileIntro}
      alt=""
      className="absolute inset-0 size-full object-cover rounded-2xl"
      style={{ opacity: introOpacity }}
    />
    <motion.img                        // 3b) about 이미지
      src={profileAbout}
      alt=""
      className="absolute inset-0 size-full object-cover rounded-2xl"
      style={{ opacity: aboutOpacity }}
    />
  </motion.div>
</motion.div>
```

### 레이어 책임
1. **이동 wrapper**: 위치(x, y)와 가시성(opacity 초기 fade-in)만 담당
2. **회전 wrapper**: 회전(rotateY, rotateZ)만 담당
3. **이미지 레이어**: 두 장 겹친 채 cross-fade로 swap

### 왜 분리하는가
- 4차 시도 함정: 한 wrapper에 x/y/rotateY 동시 → 사선 늘어짐
- 외부 사례 (Codrops, Motion docs) 모두 권장하는 구조
- 디버깅 시 책임 추적 가능 (이동 이상 → wrapper 1 / 회전 이상 → wrapper 2)

## 6. 키프레임값 — SSOT 1차값 그대로

```tsx
// 회전
const rotateY = useTransform(progress, [0, 0.32, 0.58, 0.82, 1], [0, -18, -34, -20, -10])
const rotateZ = useTransform(progress, [0, 0.58, 0.82, 1],       [0, 0.4, 1.2, 2.0])

// 이동 — 동시 점진적
const x = useTransform(progress, [0, 1], [coords.sx, coords.ex])
const y = useTransform(progress, [0, 1], [coords.sy, coords.ey])

// 이미지 swap — 가장 얇은 구간 (0.56~0.70)
const introOpacity = useTransform(progress, [0.56, 0.70], [1, 0])
const aboutOpacity = useTransform(progress, [0.56, 0.70], [0, 1])
```

### 조정 우선순위 (구현 후 튜닝 시)
- **swap 들킴**: rotateY 피크 -34° → -45° → -55° (회전 깊이 증가)
- **회전 어색**: 키프레임 분포만 손봄 (`[0, 0.32, 0.58, 0.82, 1]` 비율)
- **swap 너무 빠름/늦음**: opacity 구간 `[0.56, 0.70]`만 미세 조정

## 7. 정적 anchor 처리

### 데스크톱
```tsx
// IntroSection
<span ref={introRef} className="...portrait... md:opacity-0" />

// AboutSection
<div ref={aboutRef} className="...portrait... md:opacity-0" />
```

데스크톱(md+)에서는 항상 `opacity: 0`. floating card가 그 자리를 정확히 차지.

### 모바일
- `md:opacity-0`은 mobile에 적용 안 됨 → static portrait 정상 노출
- floating card는 mobile에서 미렌더 (Section 8)
- → 모바일은 기존 정적 portrait 그대로

### 왜 conditional 렌더링이 아닌 opacity인가
- DOM에서 사라지면 anchor 좌표 측정 불가
- opacity 0은 layout 유지 + anchor 위치 안정
- 측정 → 카드 위치 정확

## 8. Mobile 비활성

```tsx
const isActive = useMediaQuery("(min-width: 768px)")

if (!isActive) return null   // floating card 미렌더
```

### 이유
- 세로 스택 레이아웃에서는 morph 인상이 약해짐
- 스크롤 거리가 짧아 morph 윈도우 압축 → 어색함
- 모바일 기기 RAF 부담도 큼

### 모바일 fallback 동작
1. `MorphingPortrait` 미렌더
2. `md:opacity-0`이 적용 안 됨 → static portrait 노출
3. 사용자에겐 정적 페이지로 보임

const reduceMotion = useReducedMotion()  // framer-motion 내장

## 11. 초기 렌더 (FOUC 회피)

### 문제
- JS hydration 전: static portrait 보임
- JS hydration 후: `md:opacity-0` 적용 → static 사라짐, floating card 등장
- → 깜빡임 가능 (50~150ms)

### 해결
좌표 측정 완료 전엔 floating card도 invisible:

```tsx
const [ready, setReady] = useState(false)

useEffect(() => {
  const c = calcCoords()
  if (c) {
    setCoords(c)
    setReady(true)
  }
}, [])

return (
  <motion.div style={{ opacity: ready ? 1 : 0, ... }}>
    ...
  </motion.div>
)
```

### 시각적 흐름
1. `t=0`: HTML 도착, static portrait 보임
2. `t=hydrate`: `md:opacity-0` 적용, static 사라짐
3. `t=measure`: 좌표 측정 완료, floating card opacity 1로 fade-in
4. → 1과 3 사이에 짧은 빈 공간 가능

수용 기준: 100ms 이내라면 허용. 그 이상이면 측정 로직 최적화 필요.

## 13. 진행 시퀀스 (SSOT Phase 0~7 그대로 + 본 문서 적용)

### Phase 0 — 인프라
- `MorphProvider` + `MorphContext`
- 섹션에 ref 부착 (Intro, About)
- `<main>`에 ref 부착
- `MorphingPortrait` 마운트만, 모션 0
- 좌표 측정 + ready state 검증

검증: 카드가 intro anchor 자리에 정확히 표시. 모바일에선 미렌더.

### Phase 1 — 회전만
- `rotateY`만 활성, x/y는 sx/sy 락
- progress 매핑 동작 확인

검증: 회전이 좌우로 보임, 피크 -34° 자연스러움.

### Phase 2 — 이동만
- x/y만 활성, rotateY 0 락
- 경로 검증

검증: intro → about 자리로 부드럽게 이동.

### Phase 3 — 결합
- 회전 + 이동 동시
- wrapper 분리 구조 효과 검증

검증: 카드가 사선 늘어짐 없이 자연 morph.

### Phase 4 — 안착 자세
- 마지막 키프레임 -10° 확정
- about 자리 안착감 확인

### Phase 5 — 이미지 swap
- intro/about opacity cross-fade 활성
- 0.56~0.70 구간

검증: swap이 가장 얇은 구간에 가려져 자연스러움.

### Phase 6 — 정적 anchor 숨김
- `md:opacity-0` 활성
- 데스크톱에서 floating 1장만 보이는지 확인
- 모바일 fallback 정상 동작

### Phase 7 — 폴리시
- rotateZ 추가
- borderRadius 미세 조정
- easing/timing 튜닝

## 14. 코드 골격 (참고용)

```tsx
// src/components/morphing-portrait.tsx

import { createContext, useContext, useEffect, useRef, useState } from "react"
import {
  motion,
  useMotionValue,
  useReducedMotion,
  useScroll,
  useTransform,
} from "framer-motion"

import profileIntro from "@/assets/profile/profile-3.webp"
import profileAbout from "@/assets/profile/profile-2.webp"

const CARD_W = 280
const CARD_H = 350

type MorphCoords = {
  sx: number; sy: number; ex: number; ey: number
  introDocY: number; aboutDocY: number
}

type MorphValue = {
  introRef: React.RefObject<HTMLElement | null>
  aboutRef: React.RefObject<HTMLElement | null>
  mainRef:  React.RefObject<HTMLElement | null>
  isActive: boolean
}

const MorphContext = createContext<MorphValue | null>(null)

export function MorphProvider({ children }: { children: React.ReactNode }) {
  const introRef = useRef<HTMLElement | null>(null)
  const aboutRef = useRef<HTMLElement | null>(null)
  const mainRef  = useRef<HTMLElement | null>(null)
  const reduce   = useReducedMotion()
  const [isDesktop, setIsDesktop] = useState(false)

  useEffect(() => {
    const mq = window.matchMedia("(min-width: 768px)")
    const update = () => setIsDesktop(mq.matches)
    update()
    mq.addEventListener("change", update)
    return () => mq.removeEventListener("change", update)
  }, [])

  const isActive = isDesktop && !reduce

  return (
    <MorphContext.Provider value={{ introRef, aboutRef, mainRef, isActive }}>
      {children}
    </MorphContext.Provider>
  )
}

export const useMorphIntroRef = () => useContext(MorphContext)!.introRef
export const useMorphAboutRef = () => useContext(MorphContext)!.aboutRef
export const useMorphMainRef  = () => useContext(MorphContext)!.mainRef
export const useIsMorphActive = () => useContext(MorphContext)!.isActive

export function MorphingPortrait() {
  const ctx = useContext(MorphContext)!
  const { introRef, aboutRef, mainRef, isActive } = ctx
  const [coords, setCoords] = useState<MorphCoords | null>(null)
  const { scrollY } = useScroll()

  // 좌표 측정
  useEffect(() => {
    if (!isActive) return
    const calc = () => {
      const main  = mainRef.current?.getBoundingClientRect()
      const intro = introRef.current?.getBoundingClientRect()
      const about = aboutRef.current?.getBoundingClientRect()
      if (!main || !intro || !about) return
      setCoords({
        sx: intro.left - main.left + (intro.width  - CARD_W) / 2,
        sy: intro.top  - main.top  + (intro.height - CARD_H) / 2,
        ex: about.left - main.left + (about.width  - CARD_W) / 2,
        ey: about.top  - main.top  + (about.height - CARD_H) / 2,
        introDocY: intro.top + window.scrollY,
        aboutDocY: about.top + window.scrollY,
      })
    }
    calc()
    window.addEventListener("resize", calc)
    return () => window.removeEventListener("resize", calc)
  }, [isActive, introRef, aboutRef, mainRef])

  // 이미지 프리로드
  useEffect(() => {
    if (!isActive) return
    ;[profileIntro, profileAbout].forEach((src) => {
      const img = new window.Image()
      img.src = src
    })
  }, [isActive])

  // progress 매핑
  const progress = useTransform(() => {
    if (!coords) return 0
    const start = coords.introDocY - window.innerHeight / 2
    const end   = coords.aboutDocY - window.innerHeight / 2
    const s = scrollY.get()
    return Math.max(0, Math.min(1, (s - start) / (end - start)))
  })

  const rotateY = useTransform(progress, [0, 0.32, 0.58, 0.82, 1], [0, -18, -34, -20, -10])
  const rotateZ = useTransform(progress, [0, 0.58, 0.82, 1], [0, 0.4, 1.2, 2.0])
  const x = useTransform(progress, [0, 1], [coords?.sx ?? 0, coords?.ex ?? 0])
  const y = useTransform(progress, [0, 1], [coords?.sy ?? 0, coords?.ey ?? 0])
  const introOpacity = useTransform(progress, [0.56, 0.70], [1, 0])
  const aboutOpacity = useTransform(progress, [0.56, 0.70], [0, 1])

  if (!isActive || !coords) return null

  return (
    <motion.div
      className="pointer-events-none absolute left-0 top-0 z-30"
      style={{ width: CARD_W, height: CARD_H, x, y }}
    >
      <motion.div
        className="size-full"
        style={{ rotateY, rotateZ, transformOrigin: "center" }}
      >
        <motion.img
          src={profileIntro} alt=""
          className="absolute inset-0 size-full rounded-2xl object-cover"
          style={{ opacity: introOpacity }}
        />
        <motion.img
          src={profileAbout} alt=""
          className="absolute inset-0 size-full rounded-2xl object-cover"
          style={{ opacity: aboutOpacity }}
        />
      </motion.div>
    </motion.div>
  )
}
```

> 위 골격은 **참고용**. 실제 구현 시 phase별로 활성/비활성 토글하며 검증.
