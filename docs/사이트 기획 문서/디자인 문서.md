# 디자인 문서

이 문서는 메인 페이지의 비주얼 디자인 결정을 정리한다.
[메인 페이지 카피 초안](./메인%20페이지%20카피%20초안.md)과 짝을 이루며, `src/index.css`에 박힌 디자인 토큰의 의도와 사용 규칙을 함께 다룬다.

---

## 1. 톤·방향

**핵심 한 줄**: 차분하고 성찰적인 에디토리얼 톤. 기술이 아니라 경험과 사유가 중심. 문학적이되 흐리지 않고, 단정하되 딱딱하지 않게.

**가져옴**:
- 거대한 타이포그래피와 같은 문장 내 폰트 변주 (시그니처)
- 충분한 여백, 일관된 섹션 리듬
- 2열 split 레이아웃의 절제된 사용
- 베이지/세이지 모노톤 (따뜻함이 톤을 받쳐줌)

**거름**:
- 에이전시·테크 스타트업 화법 (모던·공격적·전환 강조)
- 글로우 카드, 거대 outlined 숫자, 큰 backdrop 글자 같은 과시적 장치
- 강한 흑백 대비 (베이지 톤으로 약화)
- 무리한 액센트 컬러

| 5 | 좌 모놀리스 UPPERCASE 헤딩 / 우 accordion | ⚠️ | accordion은 거름. 좌 큰 헤딩 / 우 본문 단락 구조만 차용 |

## 3. 디자인 토큰

`src/index.css`에 박힌 토큰의 의도와 사용 규칙. 값은 코드 우선이며, 이 문서는 의도 설명용이다.

### 3.1 색

shadcn 프리셋 `b6t6ENohc` (tweakcn) 적용. 라이트·다크 모드 모두 정의됨.

- **베이지/세이지 모노톤**, 채도 매우 낮음 (oklch chroma 0.005~0.031)
- 라이트 배경: 거의 흰색에 가까운 따뜻한 오프화이트
- 본문 텍스트: 진한 회갈색
- 라운드 base: `0.45rem` (≈7px)
- 액센트 컬러는 거의 사용하지 않음 (status dot 등 극소량 예외)

### 3.2 폰트

| 용도 | 폰트 | 로딩 | 비고 |
|---|---|---|---|
| 본문 | **Pretendard Variable** (dynamic subset) | npm `pretendard` | 한글 완전 커버, weight 45~920 |
| 헤딩 | **S-Core Dream** | jsdelivr CDN | weight 400/500/600 |
| 강조 한 단어 (선택) | 명조 italic | TBD | 시그니처 변주용. 사용 폰트 미결 |

### 3.3 타이포그래피 스케일

| 요소 | size | line-height | letter-spacing | weight |
|---|---|---|---|---|
| h1 (Display) | 56px | 1.2 | -1% | 500 |
| h2 | 40px | 1.3 | -0.5% | 500 |
| h3 | 30px | 1.3 | -0.5% | 500 |
| h4 | 22px | 1.4 | 0 | 400 |
| body | 17px | 1.75 | 0 | 400 |
| small | 14px | 1.55 | 0 | 400 |

**원칙**:
- 한국어에 음수 letter-spacing 거의 없음 (Display만 -1%)
- 본문 line-height 1.75로 한글 호흡 확보
- 본문 weight는 400 유지 (LaunchFolio의 600은 거름)

### 3.4 간격

**컨테이너**:
- `--container-max`: **1185px** (LaunchFolio와 동일)
- `--container-px`: 24px / 32px / 48px (모바일 / 태블릿 / 데스크톱)
- `--reading-max`: 640px (본문 한 줄 60~70자)

**섹션 수직 패딩** (반응형):
- 모바일: 64px
- 태블릿: 96px
- 데스크톱: 128px

**스택 (텍스트 간격)**:
- xs 8px / sm 16px / md 20px / lg 24px / xl 32px / 2xl 48px

**유틸리티 클래스**:
- `.container-portfolio` — max-width + 반응형 패딩
- `.section` — 반응형 수직 패딩
- `.reading` — 본문 max-width

### 3.5 라운드 스케일

base `--radius: 0.45rem` (≈7px)에서 파생:

| 토큰 | 비율 | 값 |
|---|---|---|
| sm | 0.6 | ≈4.3px |
| md | 0.8 | ≈5.8px |
| lg | 1.0 | ≈7.2px |
| xl | 1.4 | ≈10px |
| 2xl | 1.8 | ≈13px |
| 3xl | 2.2 | ≈16px |
| 4xl | 2.6 | ≈19px |

전반적으로 절제된 라운드. pill (rounded-full)은 nav, 작은 태그·뱃지에만.

### 3.6 쉐도우

| 토큰 | 용도 |
|---|---|
| `shadow-subtle` | 카드 기본 elevation |
| `shadow-soft` | 카드 호버 시 |

큰 드롭쉐도우 없음. 두 단계 미세 elevation만 사용.

### 3.7 트랜지션

| 토큰 | 값 | 용도 |
|---|---|---|
| `--duration-fast` | 150ms | 즉각 피드백 |
| `--duration-base` | 250ms | 일반 인터랙션 |
| `--duration-slow` | 400ms | 페이드인, 큰 변화 |
| `--ease-out-expo` | cubic-bezier(0.16, 1, 0.3, 1) | 권장. 부드러운 감속 |
| `--ease-smooth` | cubic-bezier(0.4, 0, 0.2, 1) | 일반 |

### 3.8 호버 패턴

문학적 톤은 **색 변화보다 명도·투명도 변화**가 어울린다.

| 클래스 | 동작 |
|---|---|
| `.hover-link` | opacity 1 → 0.7 + underline 페이드인 (250ms) |
| `.hover-card` | translateY 0 → -2px + shadow-subtle → soft (250ms) |
| `.hover-image` | 자식 `<img>`에 scale 1 → 1.02 + opacity 1 → 0.9 (400ms) |

### 3.9 포커스 링

키보드 포커스는 명확하게:
- `--focus-ring-width`: 2px
- `--focus-ring-offset`: 2px
- 색은 `--ring` 사용 (베이지 톤)

`:focus-visible`에 자동 적용. shadcn 컴포넌트는 자체 ring 처리 우선.

## 4. 섹션별 비주얼 디렉션

카피 문서의 6섹션 + 헤더 + 풋터 = 8 영역.

### 4.0 Header (레퍼런스 0)

- Floating pill 형태, 상단 고정 (sticky)
- 좌: 아바타 슬롯 (placeholder, 비워둠) + 이름 또는 로고
- 우: 섹션 앵커 메뉴 6개 — `시작 · 경험 · 기반 · 기록 · 방향 · 연락`
  - 클릭 시 해당 섹션으로 스크롤 이동 (앵커 ID 매핑: intro / experience / practice / archive / direction / closing)
- 우측 끝의 별도 CTA 버튼 없음 (메뉴의 "연락"이 그 역할)
- shadow-subtle, 배경은 약간 투명한 흰색 (backdrop-blur 가능)

### 4.1 Intro (레퍼런스 1) — 카피 1. Intro

- **시각 중심 split heading 구성**. 본문 카피는 About Me로 이전
- h1 = `다양한 직군` [포트레이트 카드] `다양한 창작` (3-column 그리드, 중앙 포트레이트 placeholder)
  - 좌우 단어 폰트: clamp(1.5rem, 4.5vw, 3.75rem), S-Core Dream Bold
  - 포트레이트 카드: aspect-[4/5], `clamp(220px, 24vw, 340px)`, bg-muted, "이미지 추가 예정" 표시
- 이브로우/CTA 모두 없음
- 모바일에서는 grid-cols-1로 stack (좌단어 → 카드 → 우단어)
- 시그니처 명조 italic 강조어는 About Me로 이전 (현재 미사용, 추후 적용 가능)

### 4.2 About Me (레퍼런스 3 일부 차용 + 자체 변형) — 카피 2. About Me

- **사진 없는 텍스트 중심 구성**. 본인 컨셉(카테고리 거부)과 충돌하는 stats/contact는 모두 거름
- 큰 섹션 라벨 "소개" (clamp 2.5-4rem, S-Core Dream Bold)
- 2열 split:
  - 좌: 메인 카피 h2 (clamp 1.5-2rem, font-bold)
  - 우: 보조 문장 2단락 (.reading, body 17px)
- 모바일: grid-cols-1 stack
- Duncan/Joseph 레퍼런스의 스탯·연락·소셜·CTA는 모두 거름 (이유: 본인 프로필에 맞지 않음, 6번 연락 섹션과 중복)
- 추후 시각 보강 가능: "현재 상태 라인" 또는 "작업 방식 3-column" 추가 검토

### 4.3 Narrative Experience (레퍼런스 2) — 카피 3. 나를 만든 경험

- 섹션 제목 h2 위 정렬 가능 (좌 또는 중앙)
- 4 카드 (21살 / 기술과 서비스 / 창작 / 사람과 시스템)
- **인터랙션: Sticky stacked cards** — 스크롤 진행에 따라 각 카드가 뷰포트 특정 위치에 sticky로 붙고, 다음 카드가 그 위에 겹쳐 쌓이는 구조. 카드가 뷰포트에서 사라지지 않고 적층됨
  - 구현: Pure CSS (`position: sticky` + 충분한 스크롤 길이의 외부 컨테이너)
  - 추가 transform·opacity 효과는 사용하지 않음. 차분 톤 유지
- 카드 = 평면적 텍스트 카드. 작은 카드 번호(01·02·03·04)는 borderless 라이트 회색으로 절제
- 호버: `.hover-card`
- 레퍼런스의 다크 + 글로우는 거름. 라이트 베이지 카드 + 미세 border + shadow-subtle

### 4.4 Learning and Practice (레퍼런스 3) — 카피 4. 지금 쌓고 있는 기반

- 2열 split: 좌 (이미지 placeholder 슬롯) / 우 (본문 3단락)
- **이미지 슬롯은 비워두고 추후 추가**. placeholder는 미세 border + 라운드, 가로세로 비율 정해서 레이아웃 잡아둠
- 본문은 .reading 폭 안에서, 강조 단락은 약간 굵게(weight 500)

### 4.5 Evidence and Archive (레퍼런스 4) — 카피 5. 쌓아온 기록

- 두 부분으로 나눔:
  1. 상단: 7개 이력 bullet 리스트 (.reading 폭, 미니멀)
  2. 하단: **결과물 그리드**
     - 창작: 공동 저서 / 미술 전시 / 임프라브 공연 / 단편소설 공모전
     - 웹 서비스: GAME RANK / LandingLin / FB 광고 라이브러리 도우미
- 그리드 카드 = 평면 카드. **상단에 이미지 placeholder 슬롯** + 제목 + 짧은 설명 + 외부 링크
- **이미지는 비워두고 추후 추가**. 슬롯은 16:9 또는 4:3 비율로 자리만 확보
- 데스크톱 3열, 태블릿 2열, 모바일 1열
- 호버: `.hover-card`. 외부 링크 있으면 `.hover-link`도 같이

### 4.6 Problem and Direction (레퍼런스 5) — 카피 6. 다루고 싶은 문제

- 2열 split: 좌 큰 h2 헤딩 / 우 본문 3단락
- **레퍼런스 5의 좌-우 split 구조만 차용**, accordion(드롭다운)은 거름
- **좌측 헤딩 sticky 적용** — 우측 본문이 스크롤되는 동안 좌측 헤딩이 화면 좌측에 머묾 (`position: sticky`)

### 4.7 Closing (레퍼런스 6) — 카피 7. Closing

- 단순한 인사 단락 (.reading)
- 연결 문장 + 이메일 한 줄 (`mailto:` 링크)
- 좌 이미지 placeholder 슬롯 (선택). **비워두고 추후 추가**
- 폼 거름

### 4.8 Footer (레퍼런스 7)

- 미니멀: 이메일 / 메뉴 (소수) / 카피라이트
- 좌측 정보 grid, 우측 또는 하단 카피라이트
- 거대 backdrop 이름 글자 거름
- 다크 배경 거름 → 베이지 베이스 유지하되 본문 영역과 약간 다른 톤(살짝 진한 베이지)

## 5. 구현 순서

빠른 톤 검증을 우선하는 순서:

1. **0 Header + 1 Intro + 2 About Me** — 상단 골격 + 톤 검증 ✅
2. (확인 후 진행)
3. **7 Closing + 8 Footer** — 하단 골격 완성, 단순
4. **3 Narrative Experience** — 카드 디자인 패턴 확정
5. **4 Learning and Practice** — 2열 split 패턴 확정
6. **5 Evidence and Archive** — 그리드 패턴
7. **6 Problem and Direction** — 마무리

**검증 지점**: 1번까지만 작성 후 사용자 확인 → 톤 OK면 다음 단계. 톤 보정 필요하면 토큰 조정.

## 6. 결정 사항 (확정)

- **Hero CTA**: 없음
- **다크 모드**: 제공하지 않음 (CSS 토큰은 shadcn 프리셋 그대로 두되 토글 UI 만들지 않음)
- **액센트 컬러**: 사용하지 않음. 베이지 모노톤 끝까지 유지
- **명조 italic 폰트**: **마루부리** (Naver Maruburi). Display·헤딩 안의 강조어에 사용
- **이미지·외부 링크**: 본문에 이미지나 외부 링크가 들어갈 자리는 **placeholder 슬롯으로 디자인하고 비워둠** (추후 추가)
  - 이미지 적용 위치: Learning and Practice 좌측, Evidence and Archive 카드 상단, Closing 좌측, Header 좌측 아바타
  - 외부 링크: Evidence and Archive 카드의 "외부 링크" 자리 (책·전시·공연·공모전·웹 서비스)
- **섹션 2 (About Me) 추가**: 인트로 본문 카피를 분리해 별도 섹션으로 이전. 사진 없는 텍스트 중심 구성. Duncan/Joseph의 stats/contact/social/CTA는 모두 거름 (본인 컨셉과 충돌)
- **섹션 3 (구 2) 인터랙션**: Sticky stacked cards (Pure CSS sticky, 추가 transform 효과 없음)
- **섹션 6 (구 5) 구조**: 레퍼런스 5의 좌-우 split만 차용, accordion 없음. **좌측 헤딩 sticky 적용**
- **헤더 메뉴**: 한글 7개 2자 — `시작 · 소개 · 경험 · 기반 · 기록 · 방향 · 연락`
- **GitHub Pages 배포**: 기본 GitHub Pages URL 사용, 커스텀 도메인 없음
