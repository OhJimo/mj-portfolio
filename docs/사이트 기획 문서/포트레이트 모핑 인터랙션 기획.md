# 포트레이트 모핑 인터랙션 기획

## 문서 목적

메인 페이지 인트로의 포트레이트 이미지가 스크롤에 따라 소개 섹션 우측으로 이동하며 3D 틸트로 전환되는 인터랙션의 동작 사양과 구현 방향을 정리한다.

## 컨셉

인트로 분할 헤딩 가운데에 놓인 포트레이트 이미지가 스크롤에 따라 소개 섹션 우측 자리로 이동한다. 이동 중 3D 원근 틸트가 비단조 곡선으로 적용되고 (중간에 피크, 끝에서 완화), 후반 구간에서 이미지 콘텐츠가 다른 사진으로 크로스페이드된다.

레퍼런스: `docs/섹션 1,2 이미지 애니메이션 이미지/1.png ~ 4.png`

## 레퍼런스 이미지 분석

### 키프레임별 관찰

| 이미지 | 스크롤 시점 | 위치 | rotateY (관찰) | 이미지 콘텐츠 |
|---|---|---|---|---|
| 1.png | 0% (히어로 안착) | 중앙 | 0° (평면) | portrait |
| 2.png | ~40% (전환 피크) | 중앙~약간 우측 | -28° ~ -32° (강함) | portrait |
| 3.png | ~75% (크로스페이드 완료) | 우측 | -22° ~ -25° (강함) | workspace (=about 이미지) |
| 4.png | 100% (about 안착) | about 우측 | -10° ~ -15° (약함) | portrait (또는 다른 portrait) |

### 핵심 패턴

1. **위치 이동은 후반에 집중**: 이미지 1→2는 거의 제자리에서 틸트만 진행, 이미지 2→3에서 우측 이동이 본격화.
2. **틸트 곡선이 비단조**: 0° → 피크(-28°) → 완화(-15°). 중간이 가장 극적이고 끝에선 안정감 있게 정착.
3. **크로스페이드가 핵심**: 이미지 2→3에서 portrait가 다른 사진(workspace)으로 완전히 교체됨. 이미지 3→4에서 다시 portrait로 (레퍼런스는 3-state, 우리는 2-state로 단순화).
4. **360° 회전 없음**: 회전축은 Y축(원근 틸트)뿐이고, Z축 2D 회전은 미세함(2° 이하).

## 동작 시나리오

### 스크롤 진행도 정의

- 진행도 0: 페이지 최상단 (`scrollY = 0`)
- 진행도 1: 소개 섹션 anchor의 중심이 viewport 중앙에 도달한 시점

### 단계별 변화 (비단조 틸트 + 후반 위치 이동)

| 진행도 | 위치 진행 | rotate (Z) | rotateY | intro opacity | about opacity |
|---|---|---|---|---|---|
| 0.0 | 0% (인트로 중앙) | 0° | 0° | 1.0 | 0.0 |
| 0.4 | 10% (거의 제자리) | 0.5° | **-28°** (피크) | 1.0 | 0.0 |
| 0.6 | 50% (이동 진행) | 1° | -25° | 1.0 | 0.0 |
| 0.8 | 85% (about 근접) | 1.5° | -22° | 0.3 | 0.7 |
| 1.0 | 100% (about 우측 안착) | 2° | **-15°** (완화) | 0.0 | 1.0 |

### 보간 구간

- **위치 (x, y)**: 진행도 0.0~0.4는 거의 정지, 0.4~1.0에서 완만한 곡선으로 이동. easeInOut 또는 다중 keyframe.
- **rotateY**: 0→-28°→-15° 비단조 곡선. 다중 keyframe 사용.
- **rotate (Z)**: 0→2° 단조 증가.
- **크로스페이드**: 진행도 0.7~1.0 구간에서 intro→about 교차. 위치/틸트가 거의 안착하는 시점에 콘텐츠 교체.

## 시각 사양

### Anchor 크기 / 위치

- 인트로 anchor 클래스: `mx-auto block aspect-[4/5] w-[clamp(220px,24vw,340px)]`
- about anchor 클래스: 동일 사이즈 + `md:-mt-6 lg:-mt-10` (기존 조정 유지)
- 두 anchor는 동일 사이즈, 따라서 width/height 보간 불필요

### Transform

초기 상태 (진행도 0):

```
perspective: 1400px
rotate: 0deg
rotateY: 0deg
border-radius: 16px (rounded-2xl)
```

최종 상태 (진행도 1):

```
perspective: 1400px
rotate: 2deg
rotateY: -18deg
border-radius: 32px (rounded-[2rem])
transform-origin: center center
```

### 이미지 소스

- intro 이미지: `https://picsum.photos/seed/mj-portrait/600/750` (개발용 placeholder, 추후 실제 사진으로 교체)
- about 이미지: `https://picsum.photos/seed/mj-about/600/750` (개발용 placeholder)

## 기술 사양

### 컴포넌트 구조

```
<ReactLenis>
  <PortraitMorphProvider>
    <Header />
    <main>
      <IntroSection />        ← intro anchor (빈 공간 + ref)
      <AboutSection />        ← about anchor (빈 공간 + ref)
      ...
      <MorphingPortrait />    ← 단일 floating 이미지 (absolute, main 기준)
    </main>
  </PortraitMorphProvider>
</ReactLenis>
```

### 라이브러리

- `framer-motion` — `useScroll`, `useTransform`, `motion.div`, `motion.img`
- `lenis` — 이미 통합됨, useScroll이 native scroll 이벤트를 청취하므로 자동 호환

### 좌표 측정

- `useLayoutEffect` 안에서 `getBoundingClientRect()` 호출
- viewport Y → document Y 변환: `rect.top + window.scrollY`
- 모핑 div는 `<main>` 내부에 `position: absolute` (main은 document (0,0) 시작)

### scrollEnd 계산

```ts
scrollEnd = aboutDocY + aboutHeight / 2 - viewportHeight / 2
```

soft anchor 중심이 viewport 중심에 오는 스크롤 위치.

### Ref 안정화

- ref callback을 `useCallback(el => { ref.current = el }, [])` 으로 메모이즈
- 매 렌더마다 새 함수가 생기는 churn 방지 → ResizeObserver/측정 안정화

### 재측정 트리거

- 윈도우 리사이즈
- ResizeObserver (anchor 크기 변경)
- 이미지 로드 (`onLoad`)
- 폰트 로드 후 (setTimeout 100ms, 500ms 안전망)

## 엣지 케이스

### 모바일 (`< md` = 768px 미만)

- 그리드가 단일 컬럼이라 morph 경로가 수직선상으로 압축됨
- **모바일에선 morph 비활성화**, intro/about anchor 각각에 정적 이미지 표시
- `matchMedia("(min-width: 768px)")` 로 활성/비활성 동적 토글

### 첫 페인트 플래시

- 측정 완료 전 `bounds.ready=false` → morph 안 그림
- intro anchor에 정적 이미지를 placeholder로 두어 빈 공간 방지
- 측정 완료 후 anchor 이미지는 `visibility: hidden` 처리 (레이아웃 유지하며 morph가 그 위치에서 시작)

### 이미지 로딩 지연

- picsum 이미지가 늦게 로딩되더라도 anchor는 explicit width + aspect-ratio로 사이즈 유지
- `<img onLoad>` 에서 measure 한 번 더 트리거

### 스크롤 위치 복원

- 새로고침 시 브라우저가 scroll 위치를 복원하면 useTransform이 현재 scrollY에 맞춰 자동 보간
- 별도 처리 불필요

### 윈도우 리사이즈 / 브레이크포인트 변경

- resize 이벤트로 measure 재실행
- 모바일 ↔ 데스크톱 전환 시 matchMedia 리스너로 morph 활성/비활성 토글

## 결정 사항

| 항목 | 결정 |
|---|---|
| 360° 회전 | ❌ — 레퍼런스 분석 결과 회전 없음. 이전 "한 바퀴 회전" 해석은 오해. |
| 3D 틸트 (rotateY) | ✅ 비단조 곡선: 0° → -28° (피크) → -15° (안착) |
| 2D 회전 (rotate) | ✅ 0° → 2° (살짝, 단조) |
| 이미지 크로스페이드 | ✅ 진행도 0.7~1.0 |
| 위치 이동 타이밍 | 후반에 집중 (0.4 이후) |
| 모바일 morph | ❌ 비활성, 정적 이미지 유지 |
| ref callback | `useCallback` 으로 안정화 |
| Anchor 사이즈 | intro/about 동일 (`clamp(220px,24vw,340px)`) |
| 이미지 개수 | 2개 (intro + about). 레퍼런스는 3-state(workspace 경유)지만 단순화. |

## 미해결 / 추후 결정

- 틸트 피크 정확한 각도: -28° vs -25° vs -32° (레퍼런스 정확 측정 필요)
- 틸트 피크 시점: 진행도 0.4 vs 0.5
- 크로스페이드 구간: 0.7~1.0 vs 0.6~1.0
- 위치 이동 곡선: easeInOut vs 다중 keyframe 직접 지정
- borderRadius 16→32 보간 vs 32 고정
- intro의 정적 placeholder 처리: `visibility: hidden` vs `opacity: 0`
- 데스크톱 활성화 브레이크포인트: md (768px) vs lg (1024px)
- 레퍼런스의 workspace 중간 상태를 우리도 둘지 (현재는 단순화하여 2-state)

## 코드 스케치

```tsx
// portrait-morph.tsx (핵심 부분)
export function MorphingPortrait() {
  const { introRef, aboutRef } = useContext(PortraitContext)
  const [bounds, setBounds] = useState({ ready: false, /* ... */ })
  const [enabled, setEnabled] = useState(false)

  useLayoutEffect(() => {
    const mq = window.matchMedia("(min-width: 768px)")
    setEnabled(mq.matches)
    mq.addEventListener("change", e => setEnabled(e.matches))
    // ... measure logic
  }, [])

  const { scrollY } = useScroll()
  const end = bounds.scrollEnd

  // 위치: 0~40%는 거의 정지, 40~100%에서 이동
  const x = useTransform(
    scrollY,
    [0, end * 0.4, end],
    [bounds.sx, bounds.sx, bounds.ex]
  )
  const y = useTransform(
    scrollY,
    [0, end * 0.4, end],
    [bounds.sy, bounds.sy, bounds.ey]
  )

  // 틸트: 비단조 곡선 (0 → -28 피크 → -15 안착)
  const rotateY = useTransform(
    scrollY,
    [0, end * 0.4, end],
    [0, -28, -15]
  )

  // Z축 회전: 단조 (0 → 2)
  const rotate = useTransform(scrollY, [0, end], [0, 2])

  // 모서리: 16 → 32
  const radius = useTransform(scrollY, [0, end], [16, 32])

  // 크로스페이드: 70% ~ 100%
  const introOp = useTransform(scrollY, [end * 0.7, end], [1, 0])
  const aboutOp = useTransform(scrollY, [end * 0.7, end], [0, 1])

  if (!enabled || !bounds.ready) return null

  return (
    <motion.div
      className="pointer-events-none absolute left-0 top-0 z-10 overflow-hidden"
      style={{
        x, y,
        width: bounds.sw,
        height: bounds.sh,
        rotate,
        rotateY,
        borderRadius: radius,
        transformPerspective: 1400,
      }}
    >
      <motion.img
        src=".../mj-portrait/..."
        className="absolute inset-0 h-full w-full object-cover"
        style={{ opacity: introOp }}
      />
      <motion.img
        src=".../mj-about/..."
        className="absolute inset-0 h-full w-full object-cover"
        style={{ opacity: aboutOp }}
      />
    </motion.div>
  )
}
```
